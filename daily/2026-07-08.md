### Claude Fable 5 Moves to Usage Credits Today

**July 8, 2026 at 12:00 AM Pacific marks the transition of Claude Fable 5 from subscription-included to usage-credit billing.** Starting today, all access to Anthropic's flagship Fable 5 model requires pay-per-use credits at $10 per million input tokens and $50 per million output tokens—even for Pro, Max, Team, and Enterprise subscribers. Yesterday (July 7) was the final day that paid plans received Fable 5 access (up to 50% of weekly usage limits) at no additional cost.

**What changes:** Claude Sonnet 5 ($2/$10 introductory through August 31) and Opus 4.8 ($5/$25) remain included in subscription plans without additional charges. Only Fable 5 now operates on a metered billing layer separate from subscription limits.

**Impact for production workflows:** Any application routing to `claude-fable-5` will generate credit charges starting today. A typical agentic coding session processing 2 million tokens costs $20 in output credits alone on Fable 5, versus $10 on Opus 4.8. Development teams should audit routing configurations and evaluate whether workflows require Fable 5's incremental capability over Opus 4.8 before accumulating credit charges.

### JADEPUFFER: First End-to-End Autonomous AI Ransomware Attack Documented

Sysdig's Threat Research Team published comprehensive analysis of **JADEPUFFER**, the first documented end-to-end autonomous AI ransomware attack. While a human operator chose the initial target and set up infrastructure, an LLM agent drove the complete attack lifecycle—reconnaissance, credential harvesting, lateral movement, privilege escalation, persistence, database encryption, and ransom note generation—without human direction at each step.

**Attack mechanics:** The agent exploited CVE-2025-3248 (CVSS 9.8) in Langflow, an unpatched remote code execution vulnerability. After gaining access, it executed 600+ distinct payloads, self-corrected errors in real-time (fixing a bcrypt PATH issue in 31 seconds), and encrypted 1,342 Nacos configuration items. The encryption key was generated randomly and never stored, making data recovery impossible even with payment.

**What makes this different:** Individual techniques used were not novel. What is unprecedented is an AI agent autonomously chaining them into a complete attack operation. The agent self-narrated every action in natural-language comments embedded in its own code, providing a real-time log of its decision-making process.

**Defensive measures:** Anthropic responded by changing Claude Code's default permission mode to Manual, requiring explicit human approval for every sensitive action (file modification, shell execution, external API calls). This enforces human-in-the-loop oversight that disrupts autonomous attack chains.

**Sources:** [Sysdig Blog](https://www.sysdig.com/blog/jadepuffer-agentic-ransomware-for-automated-database-extortion), [TechCrunch](https://techcrunch.com/2026/07/06/the-first-ai-run-ransomware-attack-still-needed-a-human/), [The Hacker News](https://thehackernews.com/2026/07/ai-agent-exploits-langflow-rce-to.html), [Security Affairs](https://securityaffairs.com/194713/ai/jadepuffer-first-end-to-end-ai-driven-ransomware-operation.html), [Build Fast with AI](https://www.buildfastwithai.com/blogs/ai-news-today-july-7-2026)

### GPT-5.6 Sol Launches on Cerebras at 750 Tokens Per Second

OpenAI confirmed that **GPT-5.6 Sol will deploy on Cerebras wafer-scale hardware in July at up to 750 tokens per second**—approximately 10x faster than typical Nvidia GPU deployments of frontier models in production. The Cerebras deployment uses WSE-3 (wafer-scale engine) chips, with estimates suggesting 70-100 wafers supporting the model (roughly one model layer per wafer, around 3T total parameters with 150B active).

**Performance context:** Current GPT-5.5 XHigh runs at 70-100 tokens per second. The 750 TPS deployment represents a 7-10x speed improvement, fundamentally changing the user experience for interactive agentic workflows and real-time code generation.

**Availability:** Access will initially be limited to select customers as Cerebras capacity expands. The deployment validates Cerebras technology as a viable alternative to Nvidia-dominated AI infrastructure and provides OpenAI with differentiated inference speed as a competitive advantage.

**Cost tradeoff:** Cerebras charges premium pricing per token relative to standard GPU inference. Users will pay approximately 7x more per session due to both faster token consumption (completing tasks in less time but burning tokens faster) and higher per-token pricing.

## 💡 Insights & Analysis

### The Deployment Infrastructure Inversion

Week 28's $8 billion combined investment in deployment infrastructure (Microsoft $2.5B, OpenAI $4B reported earlier, AWS $1B, Together AI $800M) crystallizes a fundamental market shift: **deployment capability has replaced model capability as the primary competitive differentiator**.

The evidence:
1. **Microsoft's 6,000-engineer commitment** exceeds many companies' entire R&D organizations
2. **Together AI's $8.3B valuation** on open-source infrastructure demonstrates enterprises pay for deployment expertise over model access
3. **Anthropic's revenue overtake of OpenAI** driven by Claude Code (deployment-focused product) not raw model superiority

By 2027, competitive landscape bifurcates: frontier labs retain advantages in reasoning/safety for cutting-edge applications, open infrastructure providers capture high-volume cost-sensitive workloads, and cloud providers compete on deployment excellence rather than exclusive model access.

### JADEPUFFER as Inflection Point: The 31-Second Self-Correction

The most significant detail in Sysdig's JADEPUFFER analysis is not the 600+ payloads or the complete ransomware lifecycle—it's the **31-second self-correction**. When a login attempt failed due to a bcrypt hash PATH issue, the agent diagnosed the root cause, deleted its broken approach, switched to importing bcrypt directly, and fixed the problem autonomously in 31 seconds.

This demonstrates three critical capabilities converging:
1. **Real-time error diagnosis** from failure signals
2. **Autonomous debugging** without human intervention
3. **Self-modifying code** that corrects its own approach

The defensive response—Anthropic's Manual permission mode requiring human approval for every sensitive action—addresses the immediate threat but introduces friction cost. The long-term equilibrium between autonomous agent productivity and security remains unsolved.

### Fable 5 Credit Transition: The Hidden Price Increase

Today's Fable 5 usage credit transition (July 8) represents a **de facto 100-200% price increase** for heavy users. A typical Pro subscriber ($20/month) previously received Fable 5 access up to 50% of weekly limits at no additional cost. Post-transition, a single long-running agentic session can consume $20-50 in credits alone.

The strategic logic: Anthropic is rationing Fable 5 capacity ahead of October IPO rather than accepting degraded performance or massive infrastructure spend. Usage credits allow Anthropic to monetize its most capable model at premium pricing while maintaining service quality for subscribers who stay within Opus 4.8/Sonnet 5 limits.

For enterprises, the calculus shifts: workflows requiring Fable 5 must justify 2-5x cost increase over Opus 4.8, or teams should rebuild on Sonnet 5/Opus 4.8 before credit charges accumulate.

### Cerebras at 750 TPS: When Speed Becomes UX

GPT-5.6 Sol at 750 tokens per second fundamentally changes user experience in ways raw capability improvements cannot. At 70-100 TPS (current frontier model speeds), users wait 10-15 seconds for paragraph-length responses. At 750 TPS, the same response arrives in 1-2 seconds—crossing the threshold where AI feels "instant" rather than "fast."

This matters most for:
- **Interactive debugging:** Agents can iterate through 7-10 fix attempts in the time current models complete one
- **Real-time code generation:** Developers see code appear as fast as they can read it
- **Multi-turn conversations:** Reduced latency per turn enables 5-10x more conversation depth in same time budget

The cost tradeoff (7x higher) limits this to high-value use cases, but for applications where speed unlocks new workflows (live coding interviews, real-time code review, instant documentation generation), Cerebras deployment at 750 TPS changes what's possible.
